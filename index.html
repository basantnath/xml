XML Related Questions with Solutions

Group A: Very Short Answer Questions 3 marks each

1. Why is XML namespace significant?

Solution:

XML namespaces are used to avoid element name conflicts. When using prefixes in XML, a namespace for the prefix must be defined. The namespace can be defined by an xmlns attribute in the start tag of an element.

Key points:

  • Prevents naming conflicts between elements
  • Provides a way to combine XML documents from different sources without conflicts
  • Uses URI references to identify namespaces
2. What is a well formed XML document?

Solution:

A well-formed XML document follows the syntax rules of XML:

  • Must have a root element
  • All elements must have a closing tag
  • Tags are case sensitive
  • Elements must be properly nested
  • Attribute values must be quoted
  • Special characters must be properly escaped
<?xml version="1.0" encoding="UTF-8"?>
<note>
  <to>John</to>
  <from>Jane</from>
  <body>Don't forget the meeting!</body>
</note>
3. What is #PCDATA?

Solution:

#PCDATA stands for Parsed Character DATA. It is used in XML DTDs to specify that an element contains parseable text data.

Key points:

  • Text that will be parsed by the XML parser
  • Tags inside the text will be treated as markup
  • Entities will be expanded
<!ELEMENT note (#PCDATA)>

This declares that the "note" element can contain any parseable character data.

Group B: Short Answer Questions 8 marks each

1. Differentiate between DTD and XML schema. Also mention the features of XML.

Solution:

Differences between DTD and XML Schema:

DTD XML Schema
Uses non-XML syntax Uses XML syntax
Limited data types Rich set of data types
No namespace support Full namespace support

Features of XML:

  • Human and machine readable
  • Supports Unicode
  • Platform independent
  • Can represent complex data structures
  • Separates data from presentation
2. What is DTD? Explain the creation of external DTD with example.

Solution:

DTD (Document Type Definition) defines the structure and legal elements of an XML document.

Creating an external DTD:

1. Create a file with .dtd extension (e.g., note.dtd):

<!ELEMENT note (to, from, heading, body)>
<!ELEMENT to (#PCDATA)>
<!ELEMENT from (#PCDATA)>
<!ELEMENT heading (#PCDATA)>
<!ELEMENT body (#PCDATA)>

2. Reference the DTD in your XML document:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE note SYSTEM "note.dtd">
<note>
  <to>John</to>
  <from>Jane</from>
  <heading>Reminder</heading>
  <body>Don't forget the meeting!</body>
</note>

Group C: Long Answer Questions 12 marks each

1. Define XML schema. What are the rules of writing XML? Generate the XML and XML schema for the given problem. Create a root element say class with child student and student have further three child Fname, Lname and age where Fname and Lname must be string type and age should be integer. And should be in proper sequence also.

Solution:

XML Schema is an XML-based alternative to DTD that describes the structure of an XML document.

Rules for writing XML:

  • Must have a root element
  • All elements must have a closing tag
  • XML tags are case sensitive
  • Elements must be properly nested
  • Attribute values must be quoted

XML Schema (class.xsd):

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="class">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="student" maxOccurs="unbounded">
          <xs:complexType>
            <xs:sequence>
              <xs:element name="Fname" type="xs:string"/>
              <xs:element name="Lname" type="xs:string"/>
              <xs:element name="age" type="xs:integer"/>
            </xs:sequence>
            </xs:complexType>
          </xs:element>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>

XML Document (class.xml):

<?xml version="1.0" encoding="UTF-8"?>
<class xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:noNamespaceSchemaLocation="class.xsd">
  <student>
    <Fname>John</Fname>
    <Lname>Doe</Lname>
    <age>20</age>
  </student>
  <student>
    <Fname>Jane</Fname>
    <Lname>Smith</Lname>
    <age>21</age>
  </student>
</class>
2. What are the rules for writing XML? Use XML Complex Type, Sequence and Create schema for a student having attribute with the following restrictions. Name: string type, Roll number: number, Marks: number (Ranges between 0 to 100)

Solution:

Rules for writing XML:

  • Must have a single root element
  • All elements must have closing tags
  • XML tags are case sensitive
  • Elements must be properly nested
  • Attribute values must be quoted
  • Special characters must be escaped

XML Schema with restrictions:

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="students">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="student" maxOccurs="unbounded">
          <xs:complexType>
            <xs:sequence>
              <xs:element name="name" type="xs:string"/>
              <xs:element name="rollno" type="xs:integer"/>
              <xs:element name="marks">
                <xs:simpleType>
                  <xs:restriction base="xs:integer">
                    <xs:minInclusive value="0"/>
                    <xs:maxInclusive value="100"/>
                  </xs:restriction>
                </xs:simpleType>
              </xs:element>
            </xs:sequence>
            </xs:complexType>
          </xs:element>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>
